# 1memos

1memos 是围绕随笔记录、整理、同步与待办提醒构建的个人信息应用。

## 设置

**设置首页**：
用户进入设置后的分类导航入口，只展示状态摘要与需要处理的问题，不直接承载设置项。
_避免：设置长页、设置面板_

**能力页**：
围绕一项稳定用户能力组织相关设置与操作的页面；当前包括账号与同步、记录与编辑、提醒与日历、存储与离线、外观与交互、关于与高级。
_避免：控件页、配置项集合_

## 导航

**顶层分区**：
由导航抽屉直接进入、拥有独立返回栈的主要使用区域；当前包括随笔、锦囊、待办、个人中心、归档与设置。
_避免：Tab、普通页面_

**共享转场（设计目标）**：
首页活跃分区中已有随笔卡片与编辑器内容根之间的共享边界动画；仅同栈 Home→编辑已有随笔。新建随笔、归档列表、跨顶层分区切换与系统减少动态效果开启时不启用。（ADR 0012；实现见 UI 债务收口计划）
_避免：全局共享元素、跨栈动画_


## 外观

主题不再是一组固定色板，而是 **主题描述符 = 色板 × 质感 × 密度 × 字阶 × 字体** 五根独立的轴；出厂提供四个策展风格预设，高级用户可在「外观与交互」能力页中单独调节每一根轴。（ADR 0008）

**色板**：
纯颜色决策的主题轴（主色、次色、纸面底色等）；当前五档：宣纸·朱砂、宣纸·黛蓝、玄青·荧光青、月白·中性、跟随系统动态色（Material You，仅 Android 12+）。前四种为策展色板，参与 WCAG 对比度验证。
_避免：主题、配色方案（这两个词在日常用语中混入了质感与排版含义）_

**质感**：
表面与气质决策的主题轴，决定纸面横线、朱砂竖线、印章、卡片形态与标签表现是否存在及如何呈现；当前包括文墨卷轴与清简两种。
_避免：皮肤、样式_

**密度**：
控制留白与内容松紧的主题轴，是"呼吸感"的量化旋钮；当前包括标准、宽松、紧凑三档。

**字阶**：
控制字体角色与字号层级的主题轴；当前仅标准一档（扩展预留）。

**字体**：
控制标题（含页面级标题与区块标题）所用字体家族；当前包括霞鹜文楷（全量 TTF 捆绑，OFL 授权，约 24MB）与系统字体两档。（ADR 0009）
_避免：默认系统字体（该词在 ThemeFontFamily 枚举中意为系统衬线，非无字体设置）_

**主题描述符**：
上述五根轴的完整外观组合；新增外观等于新增一个描述符数据，不需要新增组件代码。

**风格预设**：
经策展的命名主题描述符，供用户一键切换。出厂四套：文墨·朱砂（默认）、清简·月白、夜航·黛蓝、赛博·荧光青。每套副标题声明轴组合（如「宣纸朱砂 · 卷轴 · 文楷」）。
_避免：主题包_

**明暗模式**：
与主题描述符正交的模式轴，控制整个应用的浅色/深色显示；当前包括跟随系统、浅色、深色三档。

**阅读模式**：
列表与编辑器中正文可独立调节字号（小 / 标准 / 大 / 特大）与行距（紧凑 / 标准 / 宽松），不影响标题或系统组件。`ReadingConfig` + `LocalReadingConfig` 承载，外观设置页暴露。

**列表形态**：
宽屏自适应配置，控制首页时间线的列数：随窗口宽度自动单列/双列、强制单列、强制双列。宽屏阈值 `600.dp`。

**滑动操作**：
首页卡片支持左右滑动手势，动作池固定四项（加入待办、收藏、归档、置顶），左右滑分别可配置。仅归档支持撤销（Snackbar）。总开关关闭时回退纯长按。

**悬浮记录**：
通过系统悬浮窗快捷记录的入口。需用户完成「在其他应用上层显示」授权。（`QuickCaptureOverlayService`）

## 随笔生命周期

**归档**：
把随笔从主时间线收进"归档"顶层分区的可恢复收纳动作；数据完整保留，可随时取回。滑动操作中唯一支持撤销（Snackbar UNDO）。
_避免：删除、移除_

**收藏**：
把随笔收进锦囊中的文件夹；默认动作收进内置"收藏"文件夹（首次使用软创建，localFirst）。左滑收藏 / 锦囊区查看与整理。（ADR 0011）
_避免：收藏夹（避免与浏览器收藏混淆）_

**待办转换**：
滑动操作"加入待办"把随笔转为待办项；若用户无待办清单，自动创建「待办」清单并加入。

**置顶**：
Memos 原生的随笔标志（`pinned`），使随笔漂浮于时间线顶部。滑动操作可切换。

**删除**：
本应用不存在不可恢复的销毁动作；表达"从时间线移除"时只能使用归档。Memos 服务端的真删除 API 刻意不对用户暴露。（ADR 0011）
