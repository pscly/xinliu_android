## 1. 项目定位与目标

1memos 是一个面向 Android 13+ 的 memos 客户端，核心目标是：

1. **离线优先（Offline-first）**：断网可浏览本地内容，可新建/编辑记录；联网后自动同步到服务器。
2. **极速记录**：尽可能减少“打开 App -> 记录 -> 返回”的链路成本（盖章式反馈）。
3. **国风克制 UI**：宣纸/墨色/印章风格，但不牺牲可读性与操作效率。
4. **可维护、可扩展**：分层清晰（UI/领域/数据/网络/数据库/同步），便于持续加功能。

## 3. 技术栈与架构概览

### 3.2 分层设计（简化视图）

1. **UI 层（Compose）**

## 4. 离线优先数据模型（核心升级点）

离线优先的关键是**本地标识稳定**与**同步幂等**。目前采用的策略是：

1. **Room 引入本地自增主键 `localId`**
   - 本地作为主键（性能与稳定性更好）
2. **`uuid` 作为业务唯一键（本地稳定唯一标识）**
   - 附件关系等继续基于 `uuid`（避免“重键”带来的复杂迁移与丢数据风险）
3. **`serverId` 用于对齐服务端**
   - 回拉与更新时以 `serverId` 为主进行匹配，避免重复创建
4. **Room 迁移**
   - Room 版本升级并增加迁移（含历史 `PENDING -> DIRTY` 的状态转换）

一个关键踩坑点已经规避：

- **禁止 `OnConflictStrategy.REPLACE` 直接 upsert memo**
  - REPLACE 会触发“删除再插入”，可能造成外键级联（附件丢失）
  - 现采用“INSERT IGNORE + UPDATE”的方式确保附件关系不被误删

## 5. 同步策略与状态机

同步遵循“本地写入 -> 标记待同步 -> Worker 处理”的模式，UI 不直接承担复杂网络写入逻辑。

### 5.1 同步状态（SyncStatus）

当前状态机用于表达离线与同步过程：

- `LOCAL_ONLY`：仅本地存在（未绑定服务器或未上传）
- `DIRTY`：本地有改动待同步
- `SYNCING`：同步中
- `SYNCED`：已同步
- `FAILED`：同步失败（保留 `lastSyncError` 供 UI 提示/重试）

### 5.2 Worker 行为要点（稳定性）

- 创建成功后只回写 `serverId`，避免改写 `uuid`（杜绝“重键”导致的链式问题）
- 回拉时以 `serverId` 对齐更新，减少重复记录
- 附件同步遵循稳定排序（本地 sortOrder -> 服务端 attachments 顺序）

## 6. 编辑器体验与交互改造

### 6.1 “偶发白屏”根因与修复

现象：点击某些随笔后出现白屏。  
原因：导航参数未编码，服务端 uuid 可能含 `/`（例如 `memos/123`），导致路由匹配异常。  
修复：路由参数 `Uri.encode()`，读取时 `Uri.decode()`。

### 6.2 `#` 标签联想（不突兀）

- 输入 `#` 后在编辑区底部出现“标签联想”浮层（不挤压正文）
- 继续输入实时过滤；无匹配自动消失；点选标签插入并自动补空格
- 视觉：InkCard + AnimatedVisibility（淡入淡出+轻微上滑），并显示标签使用次数（可选）

### 6.3 输入法适配与丝滑优化（Android 13+）

问题：键盘弹出后底部工具/联想会被遮挡，且 UI 调整不丝滑。  
当前优化：

- 编辑页使用 `Modifier.imePadding()` 进行 IME insets 适配（更贴近系统动画路径）
- “信纸背景横线”绘制做缓存（`drawWithCache + Path`），降低输入过程 draw 开销

### 6.4 “录”与“记”的呼应（编辑页下沉）

- 编辑页的“录”印章按钮移动到右下角，和“添加图片/附件信息”同一行
- 尺寸与首页右下角“记”一致（`SealButton(size=56.dp)`），形成品牌一致性与呼应感

### 6.5 隐藏自动标签元数据行（开发者选项）

你的标签是通过 n8n/LLM 自动分析后写入“单独一行”的元数据（例如行尾含 `__Atags`）。为了让内容更干净：

- 支持按配置在**主页/查看页/编辑页**分别隐藏该行（隐藏仅影响显示，不影响保存的数据）
- 关键字判断：**行尾匹配**（忽略行尾空白；关键字前允许空白/冒号）
- 支持多关键字：逗号/空格/换行分隔（默认 `__Atags`）
- 编辑页若开启隐藏：保存时会自动把隐藏行拼回去；想修改该行时可在开发者选项里临时开启“编辑页显示该行”

### 6.6 账号登录/注册（Flow Backend 换取 Token）

根据 `1memos/to_app_plan.md`，App 新增“账号密码登录/注册”模式：

- App 只向 Flow Backend 获取 `token + server_url`，拿到后**直接连接 Memos**进行同步/写入（不经过 Backend 代理数据）
- 普通用户无感：不在 UI 中展示/配置服务器地址；填写 Token 时默认只连接固定服务器
  - 仅当“登录前已解锁开发者模式2”时，登录页才会出现“服务器地址”输入框（用于自定义）
- 入口：
  - 首次启动引导页提供“登录/同步”入口（仍可选择纯离线体验）
  - 设置页“账号状态”卡片提供“账号登录/访问令牌（Token）/退出/立即同步”操作
- 持久化：统一写入 `serverUrl + token`，额外记录 `login_mode=backend/custom` 仅用于展示与排障
- 后端地址：默认固化在 `BuildConfig.FLOW_BACKEND_BASE_URL`（测试环境：`https://2.pscly.cn:31031/`）

### 6.7 默认只看自己的历史（隐藏模式开公开/工作区）

为避免“登录后时间线混入公开/工作区内容”，App 在**账号登录**模式下默认仅展示“自己创建的笔记”（同时默认不展示公开笔记）。如确实需要浏览公开/工作区内容：

- 进入方式：设置页 → 关于 → 10 秒内连点“版本号”6次 → 输入加载密码 `pscly`
- 解锁后：可开启“查看公开/工作区笔记”
- 未解锁时：新建随笔默认提交到“工作区（PROTECTED）”，且不暴露“默认可见性（私密/工作区/公开）”选项；解锁后才提供可见性相关调试项
- 实现层：Home/个人中心热力图/编辑页标签统计均会跟随该开关做过滤

## 7. 墨迹卡片（Share Card）能力现状（plan4 核心落地）

墨迹卡片目标是：把“记录内容（文字+图片+落款）”生成一张/多张可分享的图片，做到“怎么选都好看”。

### 7.1 入口（Entry Points）

已实现双入口：

1. 详情页右上角：进入“墨迹卡片”工坊
2. 首页长按卡片：弹出“更多操作” -> “墨迹卡片”

### 7.2 工坊（Studio UI）

- 上部：实时预览（支持双指缩放）
- 下部：Tab 切换
  - `模板`：素履/墨染/宣纸/光影
  - `样式`：比例（自适应/1:1/9:16）、字号（小/中/大）、对齐（左/中）
  - `更多`：长文模式、署名、二维码开关与内容、导出进度与最近导出路径

### 7.3 长图模式（长文模式）

策略（兼顾“美观”与“稳定”）：

- 默认：截断前 140 字并提示（避免极端长文一次性导出 OOM）
- 开启“长文模式”：
  - 仅在 `自适应` 比例下生效
  - 离屏渲染先测量真实高度
  - 支持选择“分页（推荐）/单张（有上限）”，内容过长会自动回退为分页导出
  - 分享走 `ACTION_SEND_MULTIPLE`；保存相册按张数写入并提示

### 7.4 底部落款（Footer）

已支持：

- 日期：`yyyy.MM.dd`
- 署名：用户可在“更多”里填写（可选）
- 动态印章：取署名最后 1~2 个字；未填写时默认“录”
- 二维码：默认开启，可关闭/可编辑内容（默认链接为项目地址）

### 7.5 图片点缀（生活感）

- 从随笔附件中提取最多 3 张轻量缩略图（限制尺寸与数量，避免内存抖动）
- 以“拍立得”风格点缀展示，增强“日记感”

### 7.6 导出与分享（离屏渲染）

技术路径：

- Compose UI 离屏渲染到 Bitmap（`ComposeView` measure/layout -> draw 到 `Bitmap`）
- 分享：写入 `cacheDir/share_cards`，通过 `FileProvider` 分享（`FLAG_GRANT_READ_URI_PERMISSION` + `clipData` 提升兼容性）
- 保存相册：写入 `MediaStore` 到 `Pictures/1memos`（Android 13+ 无需额外存储权限）
