# 设置采用导航枢纽与六个能力页

设置首页仅作为导航枢纽，展示分类、状态摘要和需要处理的异常，不直接修改设置值；具体能力拆为“账号与同步、记录与编辑、提醒与日历、存储与离线、外观与交互、关于与高级”六个子页面。设置首页使用独立的轻量摘要 ViewModel，每个能力页分别拥有自己的 ViewModel、`UiState`、用户意图和测试面，不共享聚合全部设置的巨型 ViewModel。每个页面 ViewModel 只依赖一个面向该用户能力的深接口，由该接口隐藏设置读写、副作用编排与错误映射；不把每个动作拆成独立的浅 UseCase。该结构以用户能力而不是控件类型或代码行数划分职责，同时避免过多细粒度页面增加导航负担。
