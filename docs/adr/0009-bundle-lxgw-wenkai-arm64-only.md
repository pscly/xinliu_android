# 捆绑全量霞鹜文楷且 APK 只出 arm64-v8a

文墨感的视觉身份一半依赖中文书法字体，而绝大多数国产 Android 设备的系统衬线回退质量不可控，"书卷气"在真机上等于随机抽奖。决定捆绑**全量霞鹜文楷（LXGW WenKai，OFL 协议，不做子集化裁剪，约 20MB）**作为字阶轴上的字体档：文墨卷轴预设默认文楷标题，清简预设默认系统无衬线。

**取舍**：不做子集化是因为裁剪容易丢生僻字（笔记场景无法预料用户写什么），且 APK 体积预算放宽到 200MB 内，20MB 字体完全可承受。同时 APK 只打包 arm64-v8a 单架构，放弃 armeabi-v7a/x86 等老架构以控制体积与构建时间；现役真机与模拟器几乎全是 arm64。

**后果**：APK 体积基线上涨约 20MB；老 32 位设备无法安装；字体放大验证需覆盖文楷在固定方形印章等特殊容器中的表现。
