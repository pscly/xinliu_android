# 不引入真删除；收藏即锦囊文件夹

梳理随笔生命周期时确认：当前领域层与网络层均未接 Memos 的 DeleteMemo，应用内不存在不可恢复的销毁动作。决定**永远不引入真删除**——"从时间线移除"只有归档（可恢复），Memos 服务端的真删除 API 刻意不对用户暴露，防止手滑或同步放大造成不可挽回的数据损失。

同时，"收藏"不新建一等标志，而是**复用锦囊（Collections）文件夹体系**：左滑收藏 = 把随笔收进内置"收藏"文件夹，多个收藏入口天然就是锦囊区的多个文件夹。避免两个"收藏"体系并存造成用户困惑，也避免新增数据字段与同步逻辑。

**后果**：滑动手势动作池固定为 {加入待办, 收藏（加入锦囊）, 归档, 置顶}，左右滑动作均可在设置中自选映射；归档与删除的术语边界已写入 CONTEXT.md，后续开发不得把归档表述为删除。
