## 3. 目标 / 非目标

### 目标

1. 业务生产代码中裸 `AlertDialog` 迁移为 `PaperInkAlertDialog`（形状/容器色走纸墨令牌）。

3. designsystem 为 Seal/Chip 补 `fontScale=2` 金图契约（补齐 DESIGN §8.2 字体放大缺口中的组件级证据）。
4. `DESIGN.md` / `.ai_session` 对齐 1.16.0–1.16.2 发布事实。
5. 完整门禁 + `v1.16.2` Tag Actions 固定签名 + 正式 latest Release。

### 非目标

- 不改色板、字体包、信息架构、业务语义、数据模型。
- **不修改** `QuickCaptureOverlayService.kt`（悬浮速记冻结）。
- 不拆分 `HomeScreen` / `EditorScreen` 超大文件。
- 不引入 M7 新视觉语言（渐变/玻璃/模糊/发光）。
- 不对全部对话框做 TalkBack 全路径矩阵（只做关键确认流焦点恢复）。

### B. 关键确认焦点恢复

模式（与 AdvancedSync 一致）：

```kotlin
val focusRequester = remember { FocusRequester() }
var restoreFocus by remember { mutableStateOf(false) }
LaunchedEffect(showDialog, restoreFocus) {
    if (!showDialog && restoreFocus) {
        focusRequester.requestFocus()
        restoreFocus = false
    }
}
// 触发按钮 Modifier.focusRequester(focusRequester)
// onDismiss / confirm / dismiss 按钮：restoreFocus = true 后关闭
```

**最低覆盖**（危险/破坏性确认）：

- Home：归档/删除类确认（若有明确触发源）
- Editor：丢弃草稿 / 冲突类确认
- Todo：删除清单/任务、不可用提示的触发入口
- OneMemosApp：需用户确认的更新相关对话框（若有可挂载触发源）

Settings 中 AdvancedSync / AccountManagement **已有**恢复逻辑：只换 PaperInk，不重写。

无法稳定找到单一触发源时：**不硬加**焦点恢复，避免乱抢焦点。
